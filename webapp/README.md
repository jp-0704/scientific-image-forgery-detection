# Forgery Detection Webapp

A local Streamlit app for scientific image forgery detection. Upload an image,
get an authentic/forged label with confidence and a copy-move localisation map.

## Quick start (RTX 3050 Ti or any CUDA GPU)

```bash
cd webapp
python -m venv .venv
source .venv/bin/activate          # or .venv\Scripts\activate on Windows
pip install -r requirements.txt
```

Place trained weights into `webapp/models/`:

```
webapp/models/
  efficientnet_b4.pt
  convnext_tiny.pt
  swin_v2_tiny.pt
  swin_v2_base.pt
```

The app auto-detects which models are present — it works with any subset
(at minimum the EfficientNet baseline).

Then run:

```bash
streamlit run app.py
```

Browser opens at `http://localhost:8501`.

## Tabs

| Tab | What it does |
|-----|--------------|
| **Single Image** | Drag-drop one image → label, confidence, jet heatmap, binary mask, % forged area |
| **Batch / Folder** | Upload a ZIP or many images → progress bar → CSV download with per-image predictions and summary counts |
| **Compare Models** | One image → side-by-side prediction from every loaded model |
| **Performance** | Static comparison dashboard from `outputs/comparison/` (generated by `compare_models.ipynb`) |

## Memory notes (RTX 3050 Ti, 4 GB VRAM)

The app runs all models in **FP16** with **sequential 4-fold TTA**. Estimated
peak VRAM:

| Model | VRAM with TTA | VRAM without TTA |
|-------|---------------|------------------|
| EfficientNet-B4 | ~1.4 GB | ~0.8 GB |
| ConvNeXt-Tiny   | ~1.6 GB | ~0.9 GB |
| Swin-V2-Tiny    | ~1.8 GB | ~1.0 GB |
| Swin-V2-Base    | ~2.8 GB | ~1.5 GB |

If `cuda.OutOfMemoryError` is hit, the app **auto-degrades**:

1. Disable TTA (single forward pass)
2. Move model to CPU FP32

In the sidebar you can also turn off TTA pre-emptively.

## File layout

```
webapp/
├── app.py            Streamlit entry, tab routing
├── inference.py      Model loading + FP16 + TTA + post-processing + OOM fallback
├── arch.py           CBAM, DecoderBlock, model wrappers (5-level + 4-level UNets)
├── components.py     Streamlit UI primitives (uploaders, render helpers)
├── requirements.txt  Pinned dependencies
├── README.md         This file
└── models/           Drop trained .pt files here
```

## Troubleshooting

- **No models listed in sidebar** — verify `.pt` files are in `webapp/models/`
  with the exact filenames listed above.
- **Streamlit complains about Torch event loop** — this is harmless on first
  startup and resolves after the first model load.
- **`timm` says backbone weights download failed** — the loaded models do not
  need pretrained weights at inference time; this warning is benign.
- **Mask appears at wrong scale** — confirm the saved checkpoint includes
  `img_size: 512` in its metadata, which the training notebooks set by default.
